Other Functions

  • All functions have a length property. This property says how many parameters can be passed
  • An IIFE (immediately executed function) can be used around your code to set your code to strict when possibly doing team work and not knowing if someone else’s code is set to strict or not.
  • A function can redefine and even rewrite itself. However, we must be careful when doing this because previously assigned properties could be lost
  • We can define a function based on what methods are supported.
  • Recursive functions are functions that will continue to invoke itself until a certain condition is met.
  • Promises can be used in place of callbacks. They are said to be pending until all code leading to the promise is complete. In this state the promise is said to be unsettled. Once the operation is completed it is said to be settled. A settled promise is either:
    a. Resolved – the asynchronous operation was completed successfully
    b. Rejected – The asynchronous operation didn’t work as expected, wasn’t successfully completed, resulting in an error.
  • Asynchronous functions can be created by preceding them with async. The callback / promise / asynchronous function is preceded by the keyword await.
  • When using an asynchronous function the steps of the function should be wrapped in a try block in order to catch any errors that might be thrown.
    Each step can be written in order, and the await is used to ensure the next line of code is not ran until that part is completed (receiving a value
    from another function).
  • When using a generator function * is placed after function and before the name of the function. Yield is then used where return is generally used.
  • By using “pure functions” we are more able to create building blocks to more complex functions. This modularizes them and makes them more usable and easily understandable.
  • Higher order functions will accept another function as an argument or return another function.
  • A function has been curried when not all arguments have been supplied to the function.
  • All curried functions are high order functions because they return a function, however not all high order functions are curried.
  • A curried function allows for a more general function to be specialized to a particular situation:
